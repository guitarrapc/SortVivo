# SortVivo カード並び替え機能 仕様書

## 概要

SortCardGrid コンポーネントにドラッグ&ドロップによるソートカード並び替え機能を追加する。  
JavaScript で実装し、WASM との通信オーバーヘッドを最小化してスムーズな UX を提供する。

---

## 1. 目的・要件

### 1.1 主要目的
- ユーザーが複数の比較カードの表示順序を自由に変更できるようにする
- デスクトップ（グリッド表示）でのドラッグ操作を実装
- モバイル（カルーセル表示）では別の並び替え手段を検討（後述）

### 1.2 非機能要件
- **パフォーマンス**: 60fps を維持し、ドラッグ中のラグを最小化
- **レスポンシブ**: グリッドレイアウト（2列/3列）に対応
- **永続化**: 現在の URL 状態管理と同様に、並び替え順序を `cards` クエリパラメータに反映
- **アクセシビリティ**: 視覚的フィードバックを明確に、将来的なキーボード操作を考慮した設計

---

## 2. 動作仕様

### 2.1 ドラッグ操作（デスクトップ）

#### 2.1.1 トリガー条件
- **長押し検出**: ポインターダウンから **300ms** 経過で ドラッグモード開始
- **対象デバイス**: マウス、タッチスクリーン（Pointer Events API で統一）
- **対象要素**: `.sort-card` コンテナ全体（`.sort-card__header` など部分要素でも可）

#### 2.1.2 長押し検出中（0〜300ms）
- **視覚フィードバック**: なし（通常のホバー状態のみ）
- **動作**: タップやクリックとの誤認を防ぐため、長押し確定まで待機

#### 2.1.3 ドラッグ開始（300ms 経過時）
- **カードの状態変化**:
  - ドラッグ中のカードに `.sort-card--dragging` クラスを追加
  - `opacity: 0.3` + dashed blue outline で元の位置を視認可能に
- **ドラッグプレビュー**:
  - カードのクローン（`cloneNode(true)`）を作成し、ポインターに追従する DOM 要素として表示
  - プレビューには `.sort-card--drag-preview` クラスを付与
  - `position: fixed` で viewport 基準の絶対配置
  - **サイズは等倍**（scale 1.0）、`width`/`height` を元カードと同じ値に固定
  - `transform: translate3d(x, y, 0)` で GPU アクセラレーションを活用
  - **クリック位置オフセット保持**: ポインターダウン時のカード内相対位置（`initialOffsetX/Y`）を記録し、ドラッグ中もその位置を基準に追従（中心スナップなし）
  - プレビューには 3px solid green ボーダー + `drag-pulse` アニメーション（1.5s ease-in-out infinite）を適用
  - クローン元から継承された `transform`/`transition` を明示的にクリアしてから `document.body` に追加
  - プレビューは `pointer-events: none` で他の要素との干渉を防ぐ
- **その他の準備**:
  - カルーセルのスクロールを無効化（`carouselInterop.disableScroll(gridId)`）
  - 他の全カードに `transition: transform 200ms ease-out` を設定してアニメーション準備
  - `offsetHeight` を読み取ってリフローを強制し、transition が確実に適用されるようにする
  - ブラウザのデフォルト動作（テキスト選択など）を `preventDefault()` で抑制

#### 2.1.4 ドラッグ中（pointermove）
- **プレビュー追従**: ポインター座標に合わせて `translate3d` を更新（クリック位置オフセットを保持）
- **並び替えシミュレート**:
  - **ドロップ位置の判定アルゴリズム**:
    1. 全カードの「元の位置（CSS transform 除去後）」の中心座標を計算
    2. カーソル位置から最も近い元の位置のカードを候補として選択
    3. 候補が現在の `currentIndex` と異なる場合のみ、切り替えを検討
    4. `currentIndex` の元の位置から候補カードの元の位置への方向ベクトルを計算
    5. カーソル位置をその方向に射影し、候補カードの中心を**超えた**（投影距離 ≥ カード間距離）場合のみ `currentIndex` を更新
    6. これにより「カードの中心を超えるまで切り替わらない」直感的な UX を実現
  - 全カードの元の位置ベースで統一判定するため、`currentIndex` が変わっても座標系が一貫する
  - 他のカードを `transform: translateX/Y` でアニメーション移動（`transition: transform 200ms ease-out`）
  - transform 量は draggedIndex 基準で常に元の配置から計算（累積変換を回避）
- **スクロール境界検出**: 未実装

#### 2.1.5 ドロップ時（pointerup）
- **新しい順序の確定**:
  - `currentIndex`（ドラッグ中に更新された最終位置）を挿入位置とする
  - 元の位置と同じ場合は C# への通知をスキップ（No-op）
- **アニメーション・クリーンアップ**:
  - プレビュー要素への参照をローカル変数でキャプチャしてから `_resetDragState()` を呼び出すことで、`setTimeout` 実行時に参照が失われるのを防ぐ
  - ドラッグプレビューをフェードアウト（`opacity: 0`, 100ms linear）後に `remove()`
  - 全カードの `.sort-card--dragging` クラス、`transform`、`transition` をリセット
- **C# への通知**:
  - `dotNetRef.invokeMethodAsync('OnReorder', newOrderArray)` で新しい順序配列を送信
  - `newOrderArray` は `[0, 2, 1, 3]` のような元のインデックスの配列
- **クリーンアップ**:
  - カルーセルスクロールを再有効化（`carouselInterop.enableScroll(gridId)`）
  - `dragState` をリセット（タイマー・参照をすべてクリア）

#### 2.1.6 キャンセル条件
- **Escキー押下**: ドラッグを中止し、元の状態に復元
- **ポインターがビューポート外**: ドロップ扱いで現在位置を確定

### 2.2 モバイル対応（カルーセルモード）

#### 2.2.1 現状の課題
- カルーセルは横スクロールが主要操作であり、ドラッグ操作と干渉する
- 長押しでのドラッグは誤操作を招きやすい（スクロール意図との区別が困難）

#### 2.2.2 代替案（検討中・実装は保留）
以下のいずれかを将来的に実装する可能性がある：

**案1: 並び替えボタン UI**
- 各カードに「並び替えモード」トグルボタンを追加
- トグル ON 時、カード下部に「← →」移動ボタンを表示
- ボタンタップで隣接カードと入れ替え

**案2: 編集モードへの切り替え**
- QuickAccessPanel または PlayControlBar に「編集モード」ボタン追加
- 編集モード時はカルーセルスクロールを無効化し、ドラッグを有効化

**案3: ドラッグハンドル**
- カードの一部（例: ヘッダー左端のアイコン）を「ドラッグハンドル」とする
- ハンドル上の長押しのみドラッグを開始（他の領域は通常のスクロール）

**決定**: 現時点では実装せず、デスクトップ版の使用感を評価してから判断

## 3. 永続化仕様

### 3.1 URL クエリパラメータ
- **現状**: `?cards=Quicksort|MergeSort|HeapSort` の形式で順序を保持
- **並び替え後**: カードの新しい順序を `cards` パラメータに反映
- **実装**: `SortCardGrid.OnReorder` → `Index.UpdateContentStateUrl()` の既存フローを活用

### 3.2 localStorage
- **対象外**: カード順序は保存しない（URL が単一の真実の情報源）
- 理由: ユーザーが URL を共有した際に順序も再現されるべきため

## 4. アニメーション仕様

### 4.1 固定パラメータ
| 項目 | 値 | 備考 |
|------|-----|------|
| 長押し検出時間 | 300ms | 一般的な UX ガイドライン準拠 |
| カード移動アニメーション | 200ms | ease-out カーブ |
| プレビューフェードアウト | 100ms | linear |
| プレビューボーダーアニメーション | 1.5s | ease-in-out infinite（drag-pulse） |
| 自動スクロール速度 | 5px/frame | **未実装**（将来対応予定） |

## 5. アクセシビリティ

### 5.1 現時点の対応
- **視覚フィードバック**: ドラッグ中の透明度変化、プレビューの影表示、元の位置の点線枠
- **ARIA 属性**: 将来的に `aria-grabbed="true"`, `aria-dropeffect="move"` を検討

### 5.2 将来の拡張（キーボード操作）
- `Ctrl + ↑/↓` で選択中のカードを上下に移動
- `Tab` でカードにフォーカス、`Space` で選択状態切り替え
- スクリーンリーダーでの並び替え完了通知（`role="status"` の live region）
